1. Core Tech Stack
This is the definitive technology stack for building Daily Compass in Google Antigravity. Each choice is optimized for: rapid agentic development, executive-dysfunction-friendly UX, offline capability, and progressive enhancement across all 4 phases.
1.1 Framework & Runtime
Layer
Technology
Why This Choice
Framework
React 18+
Functional components + hooks only. Antigravity has excellent React support via its Gemini agents. Component-based architecture maps perfectly to our modular phase structure.
Language
TypeScript
Type safety catches bugs early. Antigravity agents produce better TypeScript than JavaScript because types constrain generation. Critical for complex data models (DayEntry, HabitObject, GoalObject).
Build Tool
Vite 6+
Sub-second HMR (hot module reload). Instant dev server startup. Tree-shaking for minimal bundle. Antigravity can scaffold Vite projects natively.
Package Manager
pnpm
Faster installs, strict dependency resolution, disk-efficient. Better than npm for projects with many dependencies.
1.2 Styling & Design System
Layer
Technology
Why This Choice
CSS Framework
Tailwind CSS 4
Utility-first approach means Antigravity agents can style components inline without context-switching to separate CSS files. JIT compilation = zero unused CSS.
Component Library
shadcn/ui
Copy-paste components (not npm dependency). Full control over styling. Pre-built: buttons, dialogs, dropdowns, tabs, tooltips, calendars, sliders. Saves 60%+ of UI build time.
Icons
Lucide React
Clean, consistent icon set. Tree-shakeable (only imports used icons). Works perfectly with shadcn/ui.
Animations
Framer Motion
Declarative animations for React. Layout animations, gesture support, exit animations. Makes status transitions and streak celebrations feel alive.
Fonts
Google Fonts (Fraunces + Nunito Sans + JetBrains Mono)
Fraunces for headings (warm, serif). Nunito Sans for body (friendly, readable). JetBrains Mono for timer/data. Variable fonts for performance.
1.3 State Management & Storage
Layer
Technology
Why This Choice
State Management
Zustand
Minimal boilerplate (vs Context+useReducer). Built-in middleware for persistence. DevTools support. Perfect for medium-complexity apps. One store per domain (days, habits, goals, settings).
Persistent Storage
Dexie.js (IndexedDB)
Structured database in the browser. Handles 100K+ records without lag. Query by date ranges. Better than localStorage for complex data (arrays, nested objects). Supports transactions.
Storage Fallback
localStorage
Simple key-value pairs only: settings, theme preference, last-viewed date. Never for task/habit data.
Sync Layer (Enhancement)
Firebase / Supabase
Optional cloud sync for cross-device access. See Enhancements section. Not required for Phase 1â€“4.
1.4 Data, Charts & Utilities
Layer
Technology
Why This Choice
Charts
Recharts
React-native composable charts. Bar, Line, Pie, Radar, Area. Responsive. Customizable tooltips. Lighter than Chart.js. Used for weekly/monthly dashboards.
Date Handling
date-fns
Tree-shakeable (import only what you use). Immutable. Covers: week calculations, date ranges, formatting, relative time. No moment.js bloat.
Drag & Drop
@dnd-kit/core + @dnd-kit/sortable
Modern, accessible drag-and-drop for React. Phase 3: time-block rearrangement. Phase 4: priority matrix drag between quadrants. Touch + mouse support.
CSV Export
Papaparse
Robust CSV generation and parsing. Handles edge cases (commas in text, Unicode). One-click export of all historical data.
PDF Export
jsPDF + html2canvas
Generates PDF reports from rendered HTML. Charts render as images. Used for weekly/monthly report sharing.
Form Handling
React Hook Form + Zod
Minimal re-renders. Schema validation with Zod (same schemas for storage + forms). Fast task creation forms.
Routing
React Router v7
Client-side routing: /today, /week, /month, /journal, /habits, /settings. URL-based navigation for bookmarkability.
