1. Executive Summary
Daily Compass is a browser-based personal operating system designed specifically for people experiencing executive dysfunction. Unlike generic productivity apps that assume neurotypical function, every feature is grounded in neuroscience research on ADHD, dopamine regulation, and the knowing-doing gap.
The core problem: Traditional productivity tools (Pomodoro, time-blocking, to-do lists) fail because they assume the user can initiate tasks at will. For brains with executive dysfunction, task initiation itself is the broken mechanism. Daily Compass works around this by providing external scaffolding, reducing decision overhead, building in self-compassion, and surfacing patterns that help users understand their own operating conditions.

Target User: Adult with self-suspected or diagnosed ADHD / executive dysfunction, currently job-searching, managing multiple life domains (career, health, relationships, learning) under chronic stress.
Platform: Progressive web app (React) built in Antigravity / Google environment. Browser-based, works on desktop and mobile.
Data Persistence: Local storage with cloud sync capability. All data stays private.
Design Philosophy: Low-friction, shame-free, pattern-revealing. Every interaction should take fewer decisions and less willpower than not using the app.

4.2 Analytics Engine Design
All analytics are computed client-side from stored DayEntry objects. No server-side processing required.

Key Computed Metrics:
	‚Ä¢	Completion Rate = tasks with status ‚Äúdone‚Äù / total tasks created (exclude ‚Äúdropped‚Äù from denominator if user prefers)
	‚Ä¢	Productive Time = sum of timeSpent across all tasks with status ‚Äúdone‚Äù or ‚Äúin_progress‚Äù
	‚Ä¢	Category Distribution = percentage of total tasks per category
	‚Ä¢	Energy-Productivity Correlation = Pearson correlation coefficient between daily energy and daily completion count
	‚Ä¢	Obstacle Frequency = tokenize obstacle strings, count recurring keywords/phrases, rank by frequency
	‚Ä¢	Goal Progress = matched tasks (by category or tag) / goal target
	‚Ä¢	Streak Calculation = consecutive days where at least 1 task was completed

	‚Ä¢	Step 1 ‚Äî Wins First: Display all daily wins aggregated. User adds any they missed.

5.3 Time-Blocking Design
The time-blocking view is a visual planning tool, NOT a rigid scheduler. Key design decisions:
	‚Ä¢	Granularity: 30-minute blocks (not 15 ‚Äî too granular for executive dysfunction; not 60 ‚Äî too coarse for varied tasks)
	‚Ä¢	Visual: Vertical timeline from 6 AM to 10 PM. Color-coded by category. Tasks can be dragged onto time slots.
	‚Ä¢	Flexibility: Blocks can be resized, moved, and deleted easily. Unscheduled tasks remain in a side panel.
	‚Ä¢	No alarms: No notifications or alerts. This is a visual plan, not a constraint system.
	‚Ä¢	Buffer time: Auto-suggest 15-min buffers between blocks (user can disable). Research shows ADHD brains underestimate transition time.

8.3 Executive Dysfunction UX Rules
These rules override standard UX practices where they conflict. They are derived directly from the neuroscience research:

	‚Ä¢	Rule 1 ‚Äî Fewer decisions, always. Default selections for everything. Pre-populated dropdowns. Quick-entry buttons over free text wherever possible.
	‚Ä¢	Rule 2 ‚Äî No blank pages. Every empty state has a prompt, suggestion, or example. "Add your first task" button, not an empty list.
	‚Ä¢	Rule 3 ‚Äî Celebrate completion, don‚Äôt punish incompletion. Green checkmarks animate. Streak counters celebrate. Incomplete tasks are neutral gray, never red.
	‚Ä¢	Rule 4 ‚Äî Progressive disclosure. Show the minimum viable interface first. Advanced features reveal on interaction. Don‚Äôt overwhelm.
	‚Ä¢	Rule 5 ‚Äî Undo everything. Accidental deletion, wrong status, mistaken entry ‚Äî everything is reversible. No ‚ÄúAre you sure?‚Äù dialogs (they add decisions).
	‚Ä¢	Rule 6 ‚Äî Yesterday was yesterday. Today‚Äôs view starts fresh. No carry-forward of undone tasks unless the user explicitly chose ‚ÄúMoved.‚Äù
	‚Ä¢	Rule 7 ‚Äî Shame audit every feature. Before shipping any feature, ask: "Could this make the user feel bad about themselves?" If yes, redesign.
	‚Ä¢	Rule 8 ‚Äî 2-minute entry ceiling. If the minimum daily interaction takes more than 2 minutes, the feature is too complex.
